# 메타 상점 및 영구 강화 진행 디자인 문서

## 목표
스테이지 클리어 이후에도 이어지는 성장 축을 추가해, 전투 바깥에서 누적 성장을 느낄 수 있는 메타 진행 구조를 만든다.  
이번 변경의 핵심은 `스테이지 클리어 보상 재화`, `타이틀에서 진입하는 상점`, `영구 강화`, `스테이지 진행도에 따른 티어 해금`을 하나의 일관된 흐름으로 연결하는 것이다.

## 확정 요구사항
- 스테이지를 클리어하면 메타 재화를 획득한다.
- 메타 재화는 전투 중 사용하는 `gold`와 분리한다.
- 상점은 `TitleScene`의 시작 버튼 아래에서 진입한다.
- `ThemeScene`의 기존 `Back` 버튼은 유지한다.
- `CampaignScene`에는 `TitleScene`으로 돌아가는 `Back` 버튼을 새로 추가한다.
- 상점은 범용 능력치와 타워별 능력치를 함께 판매하는 혼합형이다.
- 상점 상품은 티어형 해금 구조를 사용한다.
- 티어 해금 기준은 스테이지 진행도이며, 상위 레벨은 특정 스테이지를 클리어해야 열린다.
- 구매한 강화는 영구 적용된다.
- 영구 강화는 브라우저를 닫았다가 다시 열어도 유지되어야 하므로 `localStorage`에 저장한다.

## 디자인 원칙
- 전투 내 경제와 메타 경제는 분리한다.
- 영구 강화는 강해지는 감각을 주되 기존 전투 밸런스를 크게 깨지 않도록 소폭 누적형으로 설계한다.
- 상점은 진입 위치가 단순해야 하며, 현재 프로젝트의 씬 흐름과 충돌하지 않아야 한다.
- 영속 저장은 저장 실패나 데이터 누락에 안전해야 한다.
- 해금 규칙은 사용자가 직관적으로 이해할 수 있어야 한다.

## 권장 아키텍처
이번 기능은 기존 `campaign session`과 별개로 `meta progress` 모델을 추가하는 방식으로 설계한다.

### 1. 캠페인 세션
- 현재 프로젝트의 `selectedTheme`, `selectedStage`, `activeStage`, `clearedStages`, `scene`를 계속 담당한다.
- 역할은 “이번 플레이 흐름과 장면 전환”에 한정한다.

### 2. 메타 진행 데이터
- 영구 보상 재화
- 최고 클리어 스테이지
- 구매한 업그레이드 레벨
- 저장/불러오기 대상

### 3. 상점 카탈로그
- 판매 중인 업그레이드 정의
- 각 레벨의 가격
- 각 레벨의 효과값
- 각 레벨의 해금 스테이지

### 4. 전투 보정 계층
- 전투 시작 시 적용되는 보너스
- 타워 스탯 계산 시 적용되는 보너스
- 상점 데이터를 전투 로직에 직접 흩뿌리지 않고, 적용 계산을 한 곳에 모은다

이 분리는 현재 코드베이스의 `battle state`, `campaign session`, `stage definition` 분리 방식과 자연스럽게 맞물린다.

## 데이터 모델

### `metaProgress`
메타 진행 데이터는 아래 형태를 기본 구조로 사용한다.

```js
{
  currency: 0,
  highestClearedStage: 0,
  upgrades: {
    globalStartGold: 0,
    globalMaxLives: 0,
    globalDamageBoost: 0,
    attackTowerDamage: 0,
    attackTowerSpeed: 0,
    slowTowerEffect: 0,
    magicTowerDamage: 0,
    cannonTowerDamage: 0,
    hunterTowerSpeed: 0
  }
}
```

### 모델 책임
- `currency`
  - 상점에서 소비하는 메타 재화 총량
- `highestClearedStage`
  - 해금 판정 기준
- `upgrades`
  - 각 상품의 현재 구매 레벨

`clearedStages`는 여전히 캠페인 세션에 남기고, 상점 해금 기준은 `highestClearedStage`만 사용한다.  
이렇게 하면 “메뉴 진행 상태”와 “영구 성장 상태”가 섞이지 않는다.

## 상점 카탈로그 구조
상점 상품은 별도 카탈로그로 관리한다. 각 상품은 다음 정보를 가진다.

- `id`
- `label`
- `description`
- `category`
- `maxLevel`
- `levels`

각 `levels[n]`는 다음 정보를 가진다.

- `price`
- `value`
- `unlockStage`

예시 구조:

```js
{
  id: "globalStartGold",
  label: "보급 확장",
  description: "전투 시작 시 보유 골드를 늘린다.",
  category: "global",
  maxLevel: 3,
  levels: [
    { price: 60, value: 10, unlockStage: 1 },
    { price: 120, value: 20, unlockStage: 3 },
    { price: 210, value: 30, unlockStage: 5 }
  ]
}
```

## 업그레이드 범위

### 범용 업그레이드
- 시작 골드 증가
- 최대 라이프 증가
- 전체 타워 공격력 증가

### 타워별 업그레이드
- 공격 타워 공격력
- 공격 타워 공격 속도
- 감속 타워 감속 효율
- 마법 타워 공격력
- 캐논 타워 공격력
- 헌터 타워 공격 속도

이번 범위에서는 각 타워마다 가장 대표적인 1개 또는 2개 스탯만 강화 대상으로 삼고, 사거리나 특수 효과 증설처럼 전투 패턴을 크게 바꾸는 강화는 제외한다.

## 티어 해금 규칙
사용자는 각 상품을 한 단계씩 순차 구매한다.

### 기본 규칙
- 현재 레벨보다 정확히 다음 레벨만 구매 가능
- 재화가 충분해야 구매 가능
- 해당 레벨의 `unlockStage`를 만족해야 구매 가능
- 최대 레벨 도달 시 더 이상 구매할 수 없음

### 해금 방식
- 같은 상품 안에서 `Lv1 -> Lv2 -> Lv3`가 스테이지 진행에 따라 열린다
- 상점 전체가 한 번에 열리는 방식이 아니라, 이미 보던 상품의 상위 티어가 나중에 열리는 구조를 사용한다

이 방식은 사용자가 익숙해진 상품을 계속 성장시키는 감각을 유지하면서도, 캠페인 진행과 메타 성장이 분리되지 않도록 도와준다.

## 화면 흐름

### TitleScene
- 기존 시작 버튼은 유지한다
- 현재 시작 버튼 아래에 `Shop` 버튼을 추가한다
- 레이아웃은 사용자의 요구에 맞게 수직 흐름으로 배치한다
- `Start Campaign`은 기존처럼 `CampaignScene`으로 이동한다
- `Shop`은 신규 `ShopScene`으로 이동한다

### CampaignScene
- 기존 캠페인 맵과 스테이지 선택 흐름은 유지한다
- `TitleScene`으로 돌아가는 `Back` 버튼을 추가한다
- 이 버튼은 메타 상점 허브와 캠페인 허브를 분리하는 역할을 한다

### ThemeScene
- 기존 `Back -> CampaignScene` 흐름을 유지한다
- 상점 진입은 이 화면에 추가하지 않는다

### ShopScene
- 보유 메타 재화를 상단에 표시한다
- 범용 업그레이드 섹션과 타워별 업그레이드 섹션을 구분해 보여 준다
- 각 상품에는 현재 레벨, 다음 레벨 효과, 가격, 해금 조건을 표시한다
- 잠긴 티어는 `Stage N 클리어 시 해금` 문구와 함께 비활성 상태로 노출한다
- 최대 레벨 도달 상품은 `MAX` 상태로 표시한다
- 상점에서 나가면 `TitleScene`으로 복귀한다

## 보상 지급 흐름
메타 재화는 스테이지 클리어가 확정되는 시점에 지급한다.

### 지급 시점
- `stage-cleared`
- 최종 스테이지 완료 후 `victory`

### 지급 규칙
- 클리어한 스테이지에 대응하는 메타 보상을 `metaProgress.currency`에 더한다
- 최초 클리어 여부와 관계없이 재플레이 보상은 허용한다
- 단, 해금 진행도는 `highestClearedStage`를 기준으로만 갱신한다
- `highestClearedStage`는 더 높은 스테이지를 처음 넘었을 때만 증가한다

### 보상 스케일
- 스테이지 번호가 높을수록 보상이 완만하게 증가한다
- 급격한 보상 증가 대신 일정한 상승 폭을 사용해 메타 경제가 전투 밸런스를 압도하지 않게 한다

## 전투 반영 방식
영구 강화는 구매 즉시 저장되지만, 전투 적용은 다음 전투 시작부터 반영한다.

### 전투 시작 시 적용하는 값
- 시작 골드
- 최대 라이프

### 전투 계산 중 적용하는 값
- 전체 타워 공격력 보너스
- 타워 타입별 공격력 보너스
- 타워 타입별 공격 속도 보너스
- 감속 타워 감속 효율 보너스

### 적용 방식
- 기존 스탯 위에 소폭 가산 또는 비율 보너스를 얹는다
- 예시
  - 시작 골드 `+10 / +20 / +30`
  - 최대 라이프 `+1 / +2 / +3`
  - 전체 공격력 `+5% / +10% / +15%`
  - 특정 타워 공속 `+4% / +8% / +12%`

현재 프로젝트 밸런스를 보존하기 위해, 이번 범위에서는 공격 범위 증가나 추가 투사체처럼 체감이 큰 구조 변경형 강화는 포함하지 않는다.

## 저장 구조

### 저장 위치
- 브라우저 `localStorage`

### 저장 단위
- `metaProgress` 전체를 하나의 저장 키로 보관한다

### 저장 타이밍
- 상점 구매 직후
- 스테이지 클리어 보상 지급 직후
- 최초 로드 시 없으면 기본값 생성

### 복원 원칙
- 저장 데이터가 없으면 기본 `metaProgress`를 생성한다
- 저장 데이터 일부가 누락되면 누락 필드만 기본값으로 보정한다
- 새 업그레이드가 카탈로그에 추가되어도 기존 저장은 자동으로 0레벨 필드를 채운다

## 오류 및 예외 처리
- `localStorage` 데이터가 없으면 기본값으로 시작한다
- 저장 데이터가 손상되었거나 JSON 파싱에 실패하면 기본값으로 복구한다
- 일부 필드만 누락된 경우 전체 리셋 대신 누락 필드만 채운다
- 재화 부족 시 구매 버튼은 비활성화한다
- 잠긴 티어는 비활성화하고 해금 조건을 명시한다
- 최대 레벨 도달 상품은 `MAX`로 표시하고 구매 이벤트를 막는다
- 재플레이 보상은 허용하지만 해금 단계는 중복 상승하지 않는다

## 씬 및 모듈 책임

### 메타 진행 모듈
- 기본값 생성
- 저장/복원
- 데이터 보정
- 재화 지급
- 업그레이드 구매

### 상점 카탈로그 모듈
- 상품 정의
- 티어 가격/효과/해금값 제공

### 전투 보정 모듈
- 메타 업그레이드 레벨을 실제 전투 수치로 변환
- 초기 상태와 타워 스탯에 반영

### 씬 책임
- `TitleScene`
  - 상점과 캠페인으로 갈라지는 허브
- `CampaignScene`
  - 캠페인 탐색 허브
- `ThemeScene`
  - 선택된 스테이지 상세 및 전투 진입
- `ShopScene`
  - 메타 구매 전용 화면

## 검증 기준
- 타이틀 화면에서 시작 버튼 아래로 상점 진입 버튼이 보여야 한다
- 캠페인 화면에서 타이틀로 돌아가는 `Back` 버튼이 동작해야 한다
- 테마 화면의 기존 `Back -> CampaignScene` 흐름은 유지되어야 한다
- 스테이지 클리어 시 메타 재화가 지급되어야 한다
- 재플레이로도 재화가 지급되어야 한다
- `highestClearedStage`는 최초 최고 기록 갱신 때만 증가해야 한다
- 잠긴 티어는 구매할 수 없고 해금 조건이 표시되어야 한다
- 구매 직후 상점 UI와 저장 데이터가 즉시 갱신되어야 한다
- 브라우저를 새로고침해도 메타 재화와 구매 레벨이 유지되어야 한다
- 전투 시작 시 시작 골드와 라이프에 영구 강화가 반영되어야 한다
- 타워 스탯 계산에 범용/타워별 강화가 함께 반영되어야 한다
- 기존 캠페인 해금 흐름과 전투 진입 흐름은 깨지지 않아야 한다

## 테스트 계획
- `metaProgress` 기본값 생성 테스트
- 저장 데이터 복원 및 누락 필드 보정 테스트
- 저장 데이터 손상 시 기본값 복구 테스트
- 스테이지 클리어 보상 지급 테스트
- `highestClearedStage` 갱신 테스트
- 티어 해금 판정 테스트
- 구매 가능/불가 조건 테스트
- 상점 구매 후 재화 차감 및 레벨 증가 테스트
- 시작 골드/최대 라이프 보정 테스트
- 타워별 스탯 보정 테스트
- `Title -> Shop` 씬 이동 테스트
- `Campaign -> Title` 씬 이동 테스트
- 기존 `Theme -> Campaign` 회귀 테스트

## 비목표
- 전투 중 상점 진입
- 일회성 버프나 소모품 시스템
- 업그레이드 환불 또는 재분배 기능
- 랜덤 상점 갱신 구조
- 서버 저장 또는 계정 연동
- 전투 패턴을 크게 바꾸는 고급 특성 트리
