# 캠페인 내비게이션 UX 디자인 문서

## 목표
현재 캠페인 화면이 보드 위에 오버레이를 얹는 구조라서, 사용자가 지금 보고 있는 화면이 메뉴인지 스테이지인지 직관적으로 이해하기 어렵다.  
이번 변경의 목표는 메뉴 계층과 전투 계층을 분리해, 초기 진입은 완전한 메뉴로 시작하고 테마별 페이지와 스테이지 상세 진입 흐름을 통해 캠페인 진행을 더 명확하게 만드는 것이다.

## 현재 문제
- 앱에 들어오면 Stage 1 보드가 이미 뒤에 깔린 상태에서 오버레이만 보인다.
- 사용자는 “게임을 아직 시작하지 않았는데 왜 맵이 보이는지”를 혼동할 수 있다.
- 테마별 탐색 화면이 없어서 현재 캠페인이 스테이지 묶음인지, 선형 캠페인인지, 테마 구조인지 한눈에 보이지 않는다.
- 클리어한 스테이지를 다시 들어가는 UX가 정의되어 있지 않다.
- 현재는 전투 상태와 메뉴 상태가 강하게 분리되어 있지 않아, 탐색과 플레이의 경계가 약하다.

## 확정 범위
- 초기 화면은 전투 보드가 보이지 않는 순수 메뉴 화면으로 변경한다.
- `Start` 이후 `캠페인 메뉴`로 진입한다.
- 캠페인 메뉴에서 테마별 페이지로 이동할 수 있다.
- 각 테마 페이지는 소속 스테이지를 모두 보여 준다.
- 잠긴 스테이지도 목록에 보이도록 유지한다.
- 스테이지를 클릭하면 바로 전투가 시작되지 않고 상세 카드가 갱신된다.
- 상세 카드의 `Enter`를 눌러야 전투에 진입한다.
- 한 번 클리어한 스테이지는 언제든지 다시 진입할 수 있다.
- 이번 범위에서는 저장 방식의 영속화 여부는 별도 요구가 없으므로 런타임 기준 캠페인 진행 상태 정의에 집중한다.

## 디자인 원칙
- 메뉴와 전투는 서로 다른 장면처럼 느껴져야 한다.
- 사용자는 현재 자신이 “탐색 중인지” 또는 “플레이 중인지” 즉시 구분할 수 있어야 한다.
- 스테이지를 보는 행위와 스테이지를 시작하는 행위는 분리한다.
- 캠페인 진행도는 숨기지 않고, 잠긴 콘텐츠도 명확하게 노출한다.
- 이미 클리어한 콘텐츠는 다시 찾기 쉽고 재진입이 쉬워야 한다.

## 전체 화면 구조
앱은 크게 `메뉴 계층`과 `전투 계층`으로 나뉜다.

### 메뉴 계층
- 타이틀 메뉴
- 캠페인 메뉴
- 테마 페이지
- 스테이지 상세 카드

### 전투 계층
- 실제 보드와 HUD가 보이는 전투 화면
- 일시정지
- 스테이지 클리어
- 게임 오버
- 캠페인 클리어

핵심 규칙은 하나다.  
보드는 전투 계층에서만 보이고, 메뉴 계층에서는 보드를 보이지 않게 하거나 렌더 대상에서 제외한다.

## 화면 흐름

### 1. 타이틀 메뉴
- 앱 첫 진입 화면
- 보드는 보이지 않는다.
- 게임 제목, 짧은 설명, `Start` 버튼만 제공한다.
- `Start`를 누르면 `캠페인 메뉴`로 이동한다.

### 2. 캠페인 메뉴
- 캠페인 전체 입구 역할
- 테마 목록으로 이동하는 시작점
- 현재 진행 상황 요약을 간단히 보여 줄 수 있다.
- 여기서 특정 테마를 선택하면 `테마 페이지`로 이동한다.

### 3. 테마 페이지
- 선택한 테마에 속한 모든 스테이지를 목록으로 보여 준다.
- 잠긴 스테이지도 숨기지 않고 함께 노출한다.
- 각 스테이지는 `잠김`, `진입 가능`, `클리어` 중 하나의 상태를 가진다.
- 스테이지를 클릭해도 즉시 전투로 들어가지 않는다.
- 클릭한 스테이지 정보가 상세 카드에 반영된다.

### 4. 스테이지 상세 카드
- 테마 페이지 안의 상세 패널 또는 별도 카드 영역으로 제공한다.
- 현재 선택된 스테이지의 이름, 설명, 테마, 상태, 진입 가능 여부를 보여 준다.
- 진입 가능 상태일 때만 `Enter` 버튼이 활성화된다.
- 사용자가 `Enter`를 눌렀을 때 비로소 전투 화면으로 전환된다.

### 5. 전투 화면
- 현재처럼 보드, HUD, 타워 선택, 조작 UI를 렌더링한다.
- 이 시점부터는 `activeStage`가 실제 플레이 기준 스테이지가 된다.

### 6. 결과 화면
- `stage-cleared`, `victory`, `game-over`는 전투 이후 분기 화면이다.
- 결과 화면에서는 다음 중 일부를 제공할 수 있다.
  - 다음 스테이지로 이동
  - 같은 스테이지 재도전
  - 테마 페이지로 복귀
  - 캠페인 메뉴로 복귀

## 상태 모델
이번 UX 변경은 전투 상태와 메뉴 상태를 분리해 관리해야 한다.

### 화면 상태
- `title`
- `campaign-menu`
- `theme-page`
- `battle`
- `paused`
- `stage-cleared`
- `victory`
- `game-over`

`stage-detail`은 별도 전체 화면이 아니라 `theme-page` 안의 선택 상세 패널로 포함할 수 있다.

### 메뉴 탐색 상태
- `selectedTheme`
  - 현재 보고 있는 테마
- `selectedStage`
  - 현재 상세 카드에 표시 중인 스테이지

### 전투 상태
- `activeStage`
  - 실제 전투에 로드된 스테이지

### 캠페인 진행 상태
- `clearedStages`
  - 한 번이라도 클리어한 스테이지 번호 목록

## `selectedStage`와 `activeStage`를 분리하는 이유
- `selectedStage`는 메뉴에서 사용자가 보고 있는 후보 스테이지다.
- `activeStage`는 실제 전투에 들어간 스테이지다.
- 테마 페이지에서 스테이지를 눌렀을 때는 아직 전투 시작 전이므로 `selectedStage`만 바뀐다.
- 상세 카드에서 `Enter`를 누르는 순간 `activeStage = selectedStage`가 된다.
- 이렇게 하면 “탐색 중인 스테이지”와 “실제로 플레이 중인 스테이지”를 혼동하지 않게 된다.

## 잠금과 재진입 규칙

### 기본 해금 규칙
- 첫 테마의 첫 스테이지는 기본 해금
- 어떤 스테이지를 클리어하면 다음 스테이지 해금
- 한 테마의 마지막 스테이지를 클리어하면 다음 테마 해금

### 재진입 규칙
- `clearedStages`에 들어 있는 스테이지는 언제든 재진입 가능
- 이미 클리어한 스테이지는 테마가 달라도 다시 방문 가능해야 한다

### 잠긴 스테이지 표시
- 잠긴 스테이지도 테마 페이지에 노출한다
- 다만 클릭 시 상세 카드에서 잠금 상태를 명확히 보여 주고 `Enter`는 비활성화한다

## 화면별 책임

### 타이틀 메뉴의 책임
- 첫 인상 제공
- 전투와 분리된 시작점 제공

### 캠페인 메뉴의 책임
- 테마 진입 허브 역할
- 현재 캠페인 진척도 맥락 제공

### 테마 페이지의 책임
- 선택 가능한 스테이지와 잠긴 스테이지를 모두 보여 주는 진행도 화면 역할
- 사용자의 탐색 동작을 받는 영역

### 상세 카드의 책임
- “선택”과 “시작” 사이에 확인 단계를 제공
- 사용자가 어떤 스테이지를 들어갈지 확신하고 진입하게 만드는 역할

### 전투 화면의 책임
- 실제 플레이 진행
- 메뉴 탐색 정보가 아니라 활성 스테이지 중심 상태 사용

## UX 기대 효과
- 사용자는 게임 진입 직후 메뉴와 전투를 혼동하지 않는다.
- 캠페인 구조가 테마 단위로 명확하게 인식된다.
- 잠긴 콘텐츠도 눈에 보여 장기 진행 동기가 생긴다.
- 이미 깬 스테이지를 다시 찾고 들어가기가 쉬워진다.
- 스테이지 선택과 전투 시작이 분리되어 UX가 더 의도적으로 느껴진다.

## 비목표
- 신규 전투 시스템 추가
- 테마 전용 규칙 추가
- 저장 파일 형식 또는 영속 저장 구조 확정
- 랭크, 별점, 최고 점수 UI 추가
- 분기형 캠페인 구조 추가

## 검증 기준
- 앱 첫 진입 시 보드가 보이지 않고 메뉴만 보여야 한다.
- `Start` 이후 바로 전투가 아니라 캠페인 메뉴로 이동해야 한다.
- 테마별 화면이 분리되어 보여야 한다.
- 테마 페이지에서 잠긴 스테이지도 확인 가능해야 한다.
- 스테이지 선택만으로는 전투가 시작되지 않아야 한다.
- 상세 카드의 `Enter`로만 전투 진입이 가능해야 한다.
- 한 번 클리어한 스테이지는 언제든 다시 진입할 수 있어야 한다.
